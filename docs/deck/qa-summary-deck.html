Project Overview

BookHive is a full-stack bookstore + peer-to-peer marketplace purpose-built as a target for test automation — stable data-testid selectors, idempotent seed/reset endpoints, a 50-book fixture, two test users.

Application under test

Frontend: React 18 SPA (Vite) → Nginx on :7547
Backend: Spring Boot 3.2, Java 17 on :8080
Database: MongoDB 7
Domain: 50 books across 6 genres, cart + checkout with a 10-minute return window, peer-to-peer marketplace.

Automation stack

@civitas-cerebrum/element-interactions^0.2.8 @civitas-cerebrum/element-repository^0.1.8 @playwright/test^1.52

TypeScript 5.6 · Playwright MCP for discovery · Docker Compose test stack. The Steps API keeps raw selectors out of test code.

Starting point

Branch experimental/no-tests — the prior E2E suite and QA CI were intentionally removed in commit 658fa66. Session began with zero automated tests.

Final state

44/44 passing main suite · 7/7 failing bug-discovery repro · 10 commits · 6 findings in docs/e2e/bug-discovery-report.md.

Bug Discovery Findings

Stage-6 adversarial probing — element boundaries, flow races, context-derived. Every finding has a failing reproduction test under tests/bug-discovery/ that turns green when the bug is fixed.

IDTitleSeverityVisibility
BUG-003 Race on POST /api/orders creates ghost orders; returning them credits money never charged. Confirmed $100 → $112.99 exploit. Critical User-visible
BUG-001 Signup with a short password shows a generic "unexpected error" (unhandled 500) instead of a field-level validation message. Medium User-visible
BUG-002 POST /api/cart/items returns HTTP 500 when quantity <= 0 instead of 400. Low API-only
BUG-004 POST /api/marketplace/listings accepts arbitrary condition strings — no backend allow-list. Low API-only
BUG-005 POST /api/marketplace/listings returns HTTP 500 on zero / negative price or overlong condition. Low API-only
BUG-006 Guest page loads log a 403 from /api/auth/me — devtools noise only; UI handles correctly. No impact DOM-only

BUG-003 — the headline

Triple-clicking Checkout on a 1-book cart creates 3 orders. Only one is real; the rest are "ghosts" showing identical contents. Stock and balance each decrement once, but returning a ghost order still credits $12.99 — net positive per exploit. Two failing repro tests assert correct behaviour.

What Ships With the Suite

Tests are the deliverable — but the supporting artifacts are what make the suite survive the next change in the app.

📍 Journey map

docs/journey-map.md — 12 prioritised journeys, sentinel-bearing for resumability. Coverage checkpoint updated per cycle.

📚 App context

docs/app-context.md — testids, state transitions, gated areas. Starting knowledge base for the next session.

📖 Plain-English scenarios

docs/e2e-test-scenarios.md — 45 scenarios as a human tester would write them; reviewable by non-technical stakeholders.

🔧 API + UI helpers

helpers/api.ts seeds data authoritatively; tests drive the UI only for UX assertions. State setup uses the API for speed.

🐛 Bug reproduction suite

tests/bug-discovery/ + dedicated config. Runs in isolation; failing tests turn green when bugs are fixed.

📸 Visual evidence

docs/screenshots/ — BUG-003 ($112.99 balance after refund) + BUG-001 (generic-error signup).

Run commands

Main suite (44): npx playwright test  ·  Bug repro (7 failing): npx playwright test --config=playwright.bug-discovery.config.ts
Reset DB: curl -X POST http://localhost:8080/api/reset
